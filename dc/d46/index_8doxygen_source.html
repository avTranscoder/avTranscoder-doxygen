00021  * <h2>Main features</h2>
00022  * The library is here to provide a simple managment of the Transform process.<br/>
00023  * Main features attending are:
00024  * <ul>
00025  *  <li>process each stream without different calls</li>
00026  *  <li>specify stream process before the begin of the process</li>
00027  *  <li>minimize process: only rewrap, or transcode without transformation if possible</li>
00028  *  <li>manage different length of stream (generate silence/black)</li>
00029  *  <li>overload of each sub-component for specific integration, like proprietary wrapper</li>
00030  *  <li>multi-languages usages (C++, Java, Python)</li>
00031  * </ul>
00032  *
00033  * <h2>High Level API</h2>
00034  * <h3>Transcoder</h3>
00035  * The library provide the <b>Transcoder</b> class, which was the highest level of managment.<br/>
00036  * This classe manage progress of process for each <b>StreamTranscoder</b>.<br/>
00037  * Each <b>StreamTranscoder</b> definition can be added to the <b>Transcoder</b>.<br/>
00038  * After declaration, the process() method can process file, which call processFrame() in loop.<br/>
00039  * 
00040  * <h3>StreamTranscoder</h3>
00041  * The <b>StreamTranscoder</b> will be here to abstract call for different case as:
00042  * <ul>
00043  *  <li>simple rewrap stream</li>
00044  *  <li>transcode stream (decoding, transform and encoding)</li>
00045  *  <li>encode from dummy (silence/black or external input source), without decoding step</li>
00046  * </ul>
00047  * Each <b>StreamTranscoder</b> is able to process a frame, and in background process a rewrap or the transcode process.<br/>
00048  * 
00049  * <h2>Lowest API</h2>
00050  * <h3>InputFile</h3>
00051  * This object manage the input media, providing the analisys methods to get metadatas on wrapper and each streams.<br/>
00052  * It also the entry point to get the <b>InputStream</b> which was the coded stream.<br/>
00053  *
00054  * <h3>OutputFile</h3>
00055  * The equivalent to the <b>InputFile</b> is the <b>OutputFile</b>. Each output stream require a decalaration, needed to wrote headers of files (wrote during the beginWrap() call).<br/>
00056  * As the <b>InputFile</b> provide the <b>InputStream</b> to describe stream, the <b>OutputFile</b> as his <b>OutputStream</b> to put coded stream into the output file.<br/>
00057  * Using one <b>InputStream</b> and one <b>OutputStream</b>, each <b>DataStream</b> can be passed to provide a rewrap process.<br/>
00058  *
00059  * <h3>InputEssence</h3>
00060  * The <b>InputEssence</b> provide every time an image, from a coded stream or from an external buffer.<br/>
00061  * It's allow the possiblity to generate silence or black stream, or also switch between the coded stream and a factice (to complete stream with different durations).<br/>
00062  * The switch between source was managed inside the <b>StreamTranscoder</b> class.
00063  * 
00064  * <h3>OutputEssence</h3>
00065  * As the <b>InputEssence</b> can decode the stream, the <b>OutputEssence</b> will encode our images to provide the coded stream.<br/>
00066  * It's the encoder cofigured with some <b>Profile</b>.
00067  * 
00068  * <h3>EssenceTransform</h3>
00069  * For adapt some images to the output format, some transformation can be done.<br/>
00070  * This step will only provide pixel/sample transformation (point to point transform). No resize, resampling can be applyed in this step.
00071  * 
00072  * <h2>Low API</h2>
00073  * Each component of the Lowest API call method present in the libav/ffmpeg API. You can depend on one of these project.<br/>
00074  * Newest versions of libav/ffmpeg are recommended.<br/>
00075  * <br/>
00076  * <br/>
00077  *
